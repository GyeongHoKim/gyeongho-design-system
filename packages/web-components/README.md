# @ghds/web-components

Framework-agnostic, hand-drawn ("sketchy") web components for the GH Design
System, authored with [Lit](https://lit.dev/). Outlines are generated by
[`@ghds/sketch-core`](../sketch-core); every design value comes from
[`@ghds/tokens`](../tokens).

Components: `<gh-button>`, `<gh-card>`, `<gh-input>`.

## Install

```bash
pnpm add @ghds/web-components @ghds/tokens
```

## Usage

The components read all design values from `@ghds/tokens` **CSS custom
properties**, which inherit through the Shadow DOM boundary. Load the token
stylesheet once at the document level:

```ts
import '@ghds/tokens/css';        // defines --sys-* / --comp-* custom properties
import '@ghds/web-components';     // registers <gh-button>, <gh-card>, <gh-input>
```

```html
<gh-button variant="primary">Save</gh-button>

<gh-card elevated>
  <h3 slot="title">Sketchy card</h3>
  <p>Body content.</p>
</gh-card>

<form>
  <gh-input name="email" type="email" label="Email" required></gh-input>
  <gh-button type="submit">Submit</gh-button>
</form>
```

## Dark mode

Theming is a pure CSS-variable override — no JS re-render. `@ghds/tokens/css`
emits the dark palette under `[data-theme="dark"]` and, for users without an
explicit choice, under `@media (prefers-color-scheme: dark)`. A consumer toggles
the theme by setting the attribute on a root element:

```ts
// explicit dark
document.documentElement.setAttribute('data-theme', 'dark');
// explicit light (wins over OS preference)
document.documentElement.setAttribute('data-theme', 'light');
// follow the OS setting
document.documentElement.removeAttribute('data-theme');
```

Because the components paint with `currentColor` and `var(--sys-*)`, the new
values flow into every shadow root automatically.

## Architecture

`SketchyBase` (the Lit base class) measures the host border box via
`ResizeObserver`, fixes a deterministic PRNG `seed` **once** per instance, calls
`@ghds/sketch-core`, and injects the returned SVG path strings into an `<svg>`
overlay. Geometry regenerates only when the measured size changes, so
hover/focus state changes never make the outline shimmer. The core is
color-agnostic; strokes paint with `currentColor` driven by `sys.color` tokens.
