`MessageScroller` is currently a **React-only** component (`@ghds/react`). There is no Web Components
or React Native counterpart yet.

## Overview

`MessageScroller` is the scroll viewport for a chat log. It stacks its children vertically and, while
the reader is at the bottom, keeps the newest message in view as content grows — pausing the moment
they scroll up to read history, and resuming when they return to the bottom. Constrain its height
via `style`.

## Anatomy

A vertical flex stack inside a scroll viewport with a themed scrollbar (`comp.messageScroller.*`).
After each update it checks whether the reader is near the bottom; if so, it pins `scrollTop` to the
newest message. Fill it with `Message` rows.

No visual variants — the one behavioral toggle is `stickToBottom`.

## Accessibility

- Respect the reader's scroll position: auto-scroll only resumes when they choose to return to the
  bottom, so focus and reading are never yanked away.
- Consider a `role="log"` (or `aria-live="polite"`) on the region when messages arrive live, so new
  content is announced without stealing focus.
- The themed scrollbar uses standard `scrollbar-*` properties and degrades to the native scrollbar.
- See the <a href={withBase('/accessibility/')}>Accessibility Guide</a> for shared conventions.

## Content

N/A — the scroller has no content model of its own. It bounds and follows whatever message rows you
place inside.
