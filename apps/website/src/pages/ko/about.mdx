GHDS(GH Design System)는 일관성이나 접근성을 타협하지 않으면서 손그림(sketchy) 스타일을
유지하는 크로스 플랫폼 디자인 시스템이에요.

## 철학

- **토큰이 단일 진실 공급원** — 모든 색상, 간격, 타이포그래피, 반경, 그림자 값은
  `@ghds/tokens`에서 와요.
- **프레임워크 중립적** — React, Lit 웹 컴포넌트, React Native 모두 동일한 토큰을
  소비해요.
- **3티어 토큰** — `comp → sys → ref` 경계가 의미와 값을 분리해요.

이러한 결정 뒤의 전체 원칙 집합은 <a href={withBase('/principles/')}>디자인 원칙</a>을
참고하세요.

## 왜 손그림인가?

### 의도

스케치 렌더링은 우연이 아니라 의도적인 선택이에요:

- **친근함과 비공식성** — 흔들림은 차갑지 않고 친근하게 읽혀요.
- **인간적 따뜻함** — 손그림 선은 템플릿이 만든 것이 아니라 누군가가 만든 것처럼 느껴져요.
- **의도적으로 비기업적인 톤** — GHDS는 대부분의 컴포넌트 라이브러리가 가진 평평하고
  대체 가능한 외관을 피해요.
- **심리적 안전감** — 더 부드러운 시각적 레지스터는 오류, 빈 상태, 온보딩 순간의 날카로움을
  덜어줘요.

### 적절한 경우

- 마케팅 및 제품 표면
- 일러스트와 빈 상태 아트
- 온보딩 흐름
- 캐주얼한, 소비자 대면 맥락

### 줄여야 하는 경우

- 고신뢰 금융 UI
- 안전 중요 확인
- 법적 또는 컴플라이언스 흐름
- 조밀한 데이터 표와 엔터프라이즈 대시보드

이러한 맥락에서는 따뜻함보다 정확함과 권위가 더 중요해요 — <a href={withBase('/principles/#1-human-but-trustworthy')}>원칙 1</a>을 참고하세요.

### sketch-core의 구현 방식

`@ghds/sketch-core`는 손으로 그리는 대신 기하학을 교란해요: `packages/sketch-core/src/geometry/offset.ts`의
`offset()`이 지터를 적용하고, `packages/sketch-core/src/types.ts`의 `SketchOptions`가
모든 컴포넌트가 읽는 숫자 노브(`roughness`, `bowing`, `hachureGap`, `hachureAngle`,
`elevation`)를 노출해요. 이 노브들은 토큰 기반이며, 하드코딩되지 않아요 — <a href={withBase('/principles/#2-tokens-are-truth-exceptions-are-documented')}>원칙 2</a>와
동일한 규칙이에요.

### 거칠기 토큰 스케일 — 강도 올리기/내리기

| 강도 | roughness | bowing | hachureGap | hachureAngle | elevation |
| --- | --- | --- | --- | --- | --- |
| 없음 / 평면 | 0 | 0 | — | — | flat (0) |
| 미세 | 0.8 | 1 | tight (4) | — | — |
| 기본 | 1.4 | 2 | default (8) | -41 | raised (4) |
| 거침 / 강함 / 교차 | 2.4 | 4 (`strong`) | loose (14) | cross (41) | — |

이 값들은 참조 티어(`ref.sketch.*`)에 있어요. `sys.sketch.*`이 `default`/`raised` 값을
별칭으로 지정하여 테마에 불변하게 만들고, 각 컴포넌트의 `comp.<component>.sketch.*` 토큰
(예: `comp.button.sketch.*`, `comp.card.sketch.*`, `comp.input.sketch.*`)이 다시 `sys`
티어를 별칭으로 지정해요 — `ref`를 직접 참조하지 않아요.

특정 컴포넌트의 거칠기를 줄이려면 — 예를 들어 고신뢰 흐름의 "삭제 확인" 버튼 — 컴포넌트의
`comp.*.sketch.roughness` 토큰은 여전히 `sys`를 통해 해결되어야 하며, `ref`로 직접
건너뛰면 안 돼요 (`comp → sys → ref` 경계가 여기에도 적용돼요). 올바른 경로는 새 `sys`
티어 역할(예: `sys.sketch.roughness.subtle`, `ref.sketch.roughness.subtle` 별칭)을
추가한 다음, 해당 컴포넌트의 `comp.*.sketch.roughness` 별칭을 새 `sys` 토큰으로
재지정하고 — 이유를 주석으로 문서화하는 거예요. 숫자 거칠기 값을 하드코딩하지 말고,
`comp` 토큰이 `ref`를 직접 참조하게 하지 마세요.

## 접근성 약속

- 텍스트/배경 색상 페어링은 **WCAG 2.1 AA** 대비 기준(일반 텍스트 4.5:1, 큰 텍스트 3:1)을
  충족하며, 토큰 패키지에서 자동으로 검증돼요.
- 컴포넌트는 키보드 포커스, `aria-*` 속성, 의미 있는 라벨을 기본으로 제공해요.
- 다크 모드는 `[data-theme="dark"]` 토큰을 통해 작동하며, 사용자의 시스템 설정도
  준수해요.

키보드 상호작용, 포커스 관리, ARIA 패턴, 테스트 체크리스트는
<a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.
