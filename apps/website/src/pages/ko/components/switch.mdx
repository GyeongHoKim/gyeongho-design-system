## 개요

`Switch`는 설정을 즉시 켜거나 꺼요 — 나중에 제출되는 폼에 속하는 `Checkbox`와 달리, Switch의 효과는
바로 적용되는 것으로 기대돼요.

## 구조

실제 `<input type="checkbox" role="switch">`(React/Web Components) 또는
`accessibilityRole="switch"`를 가진 `Pressable`(React Native) 뒤에 스케치 스타일의 트랙이 그려져요.
두 번째로 작은 채워진 썸(`@ghds/sketch-core`에서 직접 계산된 타원)이 트랙 안에 렌더링되는데,
꺼졌을 때는 왼쪽, 켜졌을 때는 오른쪽에 위치해요.

## 접근성

- 세 플랫폼 모두 플랫폼의 체크박스 형 컨트롤의 실제 네이티브 토글 시맨틱을 사용해요
  (React/Web Components에서 `<input type="checkbox" role="switch">`,
  React Native에서 `accessibilityRole="switch"`) — Space/클릭 토글이 무료로 제공되고, 커스텀 keydown
  핸들러가 필요 없어요.
- React와 Web Components 모두에서, 브라우저는 `role="switch"` 재정의와 관계없이 네이티브 `checked`
  속성에서 `aria-checked`를 계산해요.
- **React Native**: 네이티브 플랫폼을 위해 `accessibilityState.checked`, 그리고 웹 타겟을 위한 직접
  `aria-checked` prop (React Native Web은 `aria-disabled`처럼 `accessibilityState`에서
  `aria-checked`를 파생하지 않아요).
- 일반적인 규칙은 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

라벨은 설정 자체의 이름이에요 ("알림 활성화"), 현재 상태나 동작이 아니에요.
