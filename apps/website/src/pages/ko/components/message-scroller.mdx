`MessageScroller`는 현재 **React 전용** 컴포넌트예요(`@ghds/react`). 아직 Web Components나 React Native 대응물은 없어요.

## 개요

`MessageScroller`는 채팅 로그의 스크롤 뷰포트예요. 자식을 세로로 쌓고, 독자가 바닥에 있는 동안 콘텐츠가 늘어나면 가장 새 메시지를 계속 보여줘요 — 독자가 히스토리를 읽으려고 위로 스크롤하는 순간 멈추고, 바닥으로 돌아오면 재개해요. `style`로 높이를 제한하세요.

## 구조

테마된 스크롤바(`comp.messageScroller.*`)가 있는 스크롤 뷰포트 안의 세로 flex 스택이에요. 매 업데이트 후 독자가 바닥 근처에 있는지 확인하고, 그렇다면 `scrollTop`을 가장 새 메시지에 고정해요. `Message` 행으로 채우세요.

## 변형 및 상태

시각적 변형은 없어요 — 유일한 동작 토글은 `stickToBottom`이에요.

## 접근성

- 독자의 스크롤 위치를 존중하세요: 자동 스크롤은 독자가 바닥으로 돌아오기를 선택할 때만 재개되므로, 포커스와 읽기가 절대 빼앗기지 않아요.
- 메시지가 라이브로 도착할 때 새 콘텐츠가 포커스를 훔치지 않고 알려지도록 영역에 `role="log"`(또는 `aria-live="polite"`)을 고려하세요.
- 테마된 스크롤바는 표준 `scrollbar-*` 속성을 사용하고 네이티브 스크롤바로 저하돼요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

  ))}
</MessageScroller>;
