`InputGroup`, `InputGroupInput`, `InputGroupAddon`은 **React 전용**이에요(`@ghds/react`). 아직 Web Components나 React Native 대응물은 없어요.

## 개요

`InputGroup`은 하나 이상의 선행 및/또는 후행 애드온 — 아이콘, 짧은 텍스트, 버튼 — 과 함께 바 텍스트 입력을 하나의 손그림 박스 안에 조합해요. 입력에 부착된 어포던스가 필요한 필드에 사용하세요: `https://` 접두사, 통화 단위, 검색 아이콘, 제출 버튼.

## 구조

자식 주위에 하나의 스케치 `SketchSurface` 박스를 그리는 flex 행이에요:

- **`InputGroupInput`** — 행을 채우도록 늘어나는, 테두리 없고 투명한 바 `<input>`.
- **`InputGroupAddon`** — 선행 또는 후행 슬롯(아이콘, 텍스트, 버튼)으로, 은은한 텍스트 색상으로 렌더돼요. 위치는 입력 앞뒤의 자식 순서로 설정돼요.

그룹 안 어디든 포커스하면 박스 외곽선이 포커스 스트로크로 전환돼요. 모든 색상, 패딩, 반경, 스케치 파라미터는 `@ghds/tokens`(`comp.inputGroup.*`)에서 와요.

## 변형 및 상태

명명된 변형은 없어요 — 대신 그룹이 `disabled`와 `invalid` 불린을 가지고, 포커스에 반응해요.

## 접근성

- 그룹은 내부 컨트롤의 focus/blur를 모아 하나의 공유 포커스 외곽선을 그려요; 자체는 인터랙티브 대상이 아니므로 role을 갖지 않아요.
- `InputGroupInput`을 통해 필드에 라벨을 지정하세요(`aria-label` 또는 연결된 `<label>`) — 애드온은 라벨을 대체하지 않는 다소 장식적인 텍스트예요.
- 그룹에 `disabled`를 설정하면 컨텍스트를 통해 `InputGroupInput`이 비활성화돼요; 개별 입력의 자체 `disabled` prop을 설정하면 그게 우선해요.
- 상태가 그려지기만 하지 않고 알려지도록 `invalid`를 `aria-invalid` / 오류 메시지와 함께 사용하세요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

애드온은 짧게 유지하세요 — 접두사(`https://`, `@`), 단위(`USD`, `/ mo`), 아이콘, 또는 작은 버튼 하나. 긴 애드온 텍스트는 입력 값과 공간을 다퉈요.
