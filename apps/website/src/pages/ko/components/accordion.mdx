## 개요

`Accordion`은 접을 수 있는 섹션들을 세로로 쌓아, 사람들이 필요한 콘텐츠만 펼쳐 볼 수 있게 해줘요. FAQ처럼 최대 한 개 섹션만 열려 있게 하려면 `type="single"`을, 여러 섹션이 동시에 열려 있게 하려면 `type="multiple"`을 사용하세요.

## 구조

손그림 스타일의 박스(`@ghds/sketch-core`)들을 세로로 쌓고, 각 박스는 헤더 버튼(`aria-expanded`, `aria-controls`)과 접을 수 있는 `role="region"`을 가져요. 헤더의 셰브론은 펼친 상태에서 회전해요. API는 **데이터 기반**이에요 — `{ value, label, content }` 형태의 `items` 배열을 전달하세요. 열린 상태는 값들의 집합(`string[]`)이며, 제어/비제어 방식 모두 지원해요.

## 접근성

- **React / Web Components**: 각 헤더는 `aria-expanded`와 `aria-controls`를 가진 `<button>`이며, `role="region"`을 가리키고(`aria-labelledby`로 다시 라벨링돼요). Arrow Up/Down과 Home/End로 헤더 간 포커스 이동, Enter/Space로 토글해요. Web Components는 `value-change` 이벤트를 발생시켜요.
- **React Native**: 헤더는 `accessibilityRole="button"`에 `accessibilityState.expanded`를 가져요; 탭으로 토글해요. 화살표 키 헤더 탐색은 웹 전용이에요(RN `Pressable`에는 `onKeyDown`이 없어요).
- <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

각 섹션의 짧은 헤더 라벨과 펼쳐지는 콘텐츠예요. 콘텐츠는 React/React Native에서 `ReactNode`이고, Web Components에서는 섹션 **텍스트**(`content` 문자열)예요.
