## 개요

`Tabs`는 같은 공간에서 관련된 콘텐츠 패널 사이를 전환하게 해줘요. 한 번에 하나의 탭이 활성화되고,
해당 패널이 표시되며 나머지는 숨겨져요.

## 구조

`role="tabpanel"` 영역 위에 `role="tab"` 버튼의 `role="tablist"`가 있어요. 활성 탭은 스케치 스타일의
박스(`@ghds/sketch-core`)로 채워지고 강조 텍스트 색상으로 표시돼요; 해당 패널은 `aria-controls`/
`aria-labelledby`로 연결돼요. API는 **데이터 기반**이에요 — `{ value, label, content }`의 `items` 배열을
전달하세요.

## 접근성

- **React / Web Components**: 완전한 WAI-ARIA Tabs 패턴 — `tablist` / `tab` / `tabpanel` 역할,
  `aria-selected`, `aria-controls`/`aria-labelledby`, 그리고 **루빙 tabindex**(활성 탭만 탭 순서에 있어요).
  화살표 키 / Home / End로 탭을 이동하고 활성화해요 (자동 활성화). Web Components는 `value-change` 이벤트를
  디스패치해요.
- **React Native**: 탭은 `accessibilityState.selected`를 가진 `role="tab"`이에요; 탭으로 선택해요.
  화살표 키 루빙 탐색은 React Native의 **알려진 차이**예요 (`Pressable`에 `onKeyDown`이 없어요) —
  React와 Web Component 빌드에서 사용할 수 있어요.

## 콘텐츠

탭당 짧은 라벨과 패널 콘텐츠. 패널 콘텐츠는 React/React Native에서 `ReactNode`예요;
Web Components에서는 패널 **텍스트**(`content` 문자열)예요 — 풍부한 마크업이 필요하면 자체 패널을
렌더링하고 `value-change` 이벤트로 구동하세요.
