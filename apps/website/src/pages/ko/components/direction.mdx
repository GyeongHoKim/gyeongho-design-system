React는 이를 `DirectionProvider` 컨텍스트와 `useDirection()` 훅으로 제공하고, Web Components는 `<gh-direction>` 요소를 제공해요 — 이 요소는 `dir`을 호스트에 미러링해 슬롯된 하위 트리가 이를 상속하게 해요. React Native도 동일한 `DirectionProvider` + `useDirection()`을 제공하지만, 프로바이더가 없으면 `'ltr'`이 아니라 앱의 `I18nManager` 레이아웃 방향으로 폴백해요.

## 개요

`DirectionProvider`는 읽기 방향 — `'ltr'` 또는 `'rtl'` — 을 React 컨텍스트를 통해 하위 GHDS 컴포넌트에 전파하고, `useDirection()`은 가장 가까운 프로바이더의 값을 읽어요. 순전히 동작용이에요: 각 컴포넌트가 DOM을 검사하지 않고도 JS에서 방향을 읽을 수 있게 해줘요.

## 구조

프로바이더는 **어떤 요소도 렌더하지 않아요** — 컨텍스트 값만 제공해요. CSS 논리 속성(`margin-inline-start`, `inset-inline-end`, …)이 해석되도록 문서나 감싸는 요소에 일치하는 `dir` 속성을 설정하세요; 프로바이더는 그 속성의 JS에서 읽을 수 있는 거울이에요.

## 변형 및 상태

시각적 변형이나 상태가 없어요 — 유일한 축은 프로바이더가 전달하는 `dir` 값이에요.

## 접근성

- 올바른 방향은 오른쪽에서 왼쪽으로 쓰는 언어(아랍어, 히브리어, 페르시아어)에서 실질적인 접근성 요구사항이에요 — 텍스트, 포커스 순서, 아이콘이 미러링되어야 해요.
- 프로바이더는 DOM에 `dir`을 설정하지 않아요; 보조 기술과 브라우저의 양방향 알고리즘도 방향을 얻도록 네이티브 `dir` 속성과 함께 사용하세요.
- `useDirection()`은 프로바이더가 없을 때 `'ltr'`을 반환하므로 컴포넌트에 안전한 기본값이 있어요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 라이브 데모

React와 Web Components가 아래에서 라이브 렌더돼요 — LTR과 RTL 방향 아래의 동일한 콘텐츠로, RTL 열이 미러링돼요. 각 열은 논리 속성이 해석되도록 네이티브 `dir` 속성도 지녀요. React Native는 추가 `react-native-web` 연결 없이는 브라우저에서 실행할 수 없어요(이 사이트에 설정되지 않음), 따라서 사용법은 코드 샘플로 보여줘요.

```tsx
// React Native
import { DirectionProvider, useDirection } from '@ghds/react-native/direction';
import { Text } from 'react-native';

function Price() {
  const dir = useDirection(); // falls back to I18nManager.isRTL when no provider is present
  return <Text>{dir === 'rtl' ? 'السعر' : 'Price'}</Text>;
}

<DirectionProvider dir="rtl">
  <Price />
</DirectionProvider>;
```
