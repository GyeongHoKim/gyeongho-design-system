## 개요

`Progress` 표시줄은 작업이 어느 정도 진행되었는지 알려줘요. 완료도를 측정할 수 있을 때는 `value`를 전달해서 **결정형(determinate)** 막대를 사용하고, 측정할 수 없을 때는 생략해서 **비결정형(indeterminate)** 스윕을 사용하세요. 매우 짧고 수량화할 수 없는 대기 시간에는 <a href={withBase('/components/spinner/')}>Spinner</a>를 사용하는 게 더 좋아요.

## 구조

`@ghds/sketch-core`로 윤곽선이 그려진 스케치 스타일의 둥근 레일(`comp.progress.bg.rail`) 위에 채워진 막대(`comp.progress.bg.fill`)가 올라가요. 결정형 막대의 너비는 `value / max`이고, 비결정형 막대는 레일을 가로지르며 스윕하는 고정 길이 세그먼트예요.

## 접근성

- **React**: `role="progressbar"`와 `aria-valuemin`, `aria-valuemax`, `aria-valuenow`를 사용해요
  (비결정형일 때는 value 속성을 생략해요). 접근성 이름으로 `label`을 전달하세요.
- **Web Components**: `gh-progress`는 호스트에 동일한 `role`/`aria-value*` 속성을 설정해요.
- **React Native**: `accessibilityRole="progressbar"`, `accessibilityLabel`, `accessibilityValue`를 설정해요
  (참고: `accessibilityValue`는 react-native-web에서 DOM으로 전달되지 않아서, 웹 빌드에서는
  `aria-valuenow`가 없을 수 있어요 — role과 label은 여전히 있어요).
- **동작 감소**: 비결정형 스윕은 `prefers-reduced-motion`(웹) 또는 OS "동작 줄이기" 설정(React Native)에서 숨겨져요.
- <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.
