## 개요

`Radio`는 단일 선택 집합에서 하나의 옵션을 나타내요. `RadioGroup` 안에서 사용하며, RadioGroup이
각 라디오의 체크 상태를 자체 `value`에서 파생하고 상호 배타성을 보장해요.

## 구조

실제 `<input type="radio">`(React/Web Components) 또는 `accessibilityRole="radio"`를 가진
`Pressable`(React Native) 뒤에 스케치 스타일의 링이 그려져요. 체크되면 링 안에 두 번째로 작은 채워진 점이
렌더링되는데, `@ghds/sketch-core`에서 직접 계산되며 링의 측정된 크기와 시드를 공유해요.

## 접근성

- **React**: 실제 `<input type="radio">`가 네이티브 키보드 동작을 제공해요 — 같은 `name`을 공유하는
  라디오 간에 화살표 키로 이동하고, Space/클릭으로 선택해요. `RadioGroup`이 컨텍스트를 통해 공유 `name`을
  제공해서 각 `Radio`에 반복할 필요가 없어요.
- **Web Components**: `gh-radio`는 섀도 DOM에 실제 `<input type="radio">`를 렌더링하지만,
  **같은 `name`의 상호 배타성은 커스텀 엘리먼트에 자동으로 적용되지 않아요** — `<gh-radio-group>`이
  버블링되는 `change` 이벤트를 수신하고 형제 라디오를 수동으로 체크 해제해요. `name`을 슬롯된 자식에
  전파하지 않아요 (Light DOM에는 컨텍스트 같은 메커니즘이 없어요) — 각 `<gh-radio>`에 `name`을 명시적으로
  반복하세요.
- **React Native**: `accessibilityRole="radio"` + `accessibilityState.checked`, 그리고 웹 타겟을 위한
  직접 `aria-checked` prop (React Native Web은 `aria-disabled`처럼 `accessibilityState`에서
  `aria-checked`를 파생하지 않아요).
- 일반적인 규칙은 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

라벨은 옵션 자체의 이름이에요 ("Small", "Medium", "Large"), 동작이 아니에요.

## 라이브 데모

React와 Web Components가 아래에 라이브로 렌더링돼요. 각각 자체 `RadioGroup`/`<gh-radio-group>`으로
감싸져 있어요. React Native는 추가적인 `react-native-web` 연결 없이는 브라우저에서 실행할 수 없어서,
사용법을 코드 샘플로 보여드릴게요.
