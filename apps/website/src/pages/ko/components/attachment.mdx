`Attachment`은 현재 **React 전용** 컴포넌트예요(`@ghds/react`). 아직 Web Components나 React Native 대응물은 없어요.

## 개요

`Attachment`은 파일이나 리소스를 나타내는 컴팩트한 칩이에요 — 선택적 선행 아이콘, 선택적 메타데이터(크기 등)가 있는 이름, 선택적 제거 버튼. 업로드된 파일, 이메일 첨부, 또는 사용자가 검토하고 선택적으로 제거할 수 있는 참조 리소스를 보여줄 때 사용하세요.

## 구조

손그림 박스(`@ghds/sketch-core`)가 좌에서 우로 배치해요: 선택적 `Icon`, 쌓인 이름 + `meta` 열, 그리고 — `onRemove`가 설정되면 — 닫기 아이콘이 있는 제거 `<button>`. 모든 색상, 패딩, 반경, 스케치 파라미터는 `@ghds/tokens`(`comp.attachment.*`)에서 와요.

## 변형 및 상태

명명된 변형은 없어요 — 칩은 제공하는 선택적 부분(`icon`, `meta`, `onRemove`)에 따라 형성돼요.

## 접근성

- 제거 컨트롤은 접근 가능한 라벨(`removeLabel`, 또는 기본 `Remove {name}`)을 가진 실제 `<button>`이므로, 스크린 리더 사용자가 어떤 첨부를 분리하는지 정확히 알아요.
- 선행 아이콘은 장식용이에요; 이름 텍스트가 의미를 가져요.
- 첨부 집합을 렌더할 때는 개수와 그룹핑이 전달되도록 목록 구조에 배치하세요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

`name`은 실제 파일 이름으로, `meta`는 짧고 사실적인 설명자(크기, 타입, 날짜)로 유지하세요. `meta` 안에 이름을 중복하지 마세요.
