## 개요

`Resizable`은 사용자가 패널 크기를 조정할 수 있는 분할 뷰예요 — 포인터 이벤트로 만들어졌고 서드파티 엔진이 없어요. 조정 가능한 레이아웃에 사용하세요: 내비 / 에디터 / 인스펙터 3분할, 목록-상세 분할, 미리보기 패널. `ResizablePanelGroup` 안에 `ResizableHandle`로 구분된 `ResizablePanel`을 배치하세요; 크기는 그룹의 백분율이에요.

## 구조

`direction`을 따라 배치된 flex `ResizablePanelGroup`이 `ResizableHandle`로 나뉜 `ResizablePanel`을 담아요. 각 핸들은 손그림 구분선(`@ghds/sketch-core` line)으로, `aria-valuenow` / `aria-valuemin` / `aria-valuemax`를 갖춘 포커스 가능한 `role="separator"`예요. 핸들을 드래그하거나 — 포커스한 뒤 화살표 키를 누르면 — 인접한 두 패널 사이에서 공간이 재분배돼요.

## 변형 및 상태

그룹의 `direction`(`'horizontal'` 또는 `'vertical'`)이 주축이고, 핸들은 `withHandle`로 그립을 보여줄 수 있어요.

## 접근성

- 각 핸들은 `tabIndex={0}`, `aria-valuenow`(선행 패널의 백분율), `aria-valuemin={0}`, `aria-valuemax={100}`, 그리고 분할에 맞는 `aria-orientation`을 갖춘 `role="separator"`예요 — 그래서 스크린 리더가 라이브 값을 가진 조작 가능한 분할선으로 알려요.
- 키보드 리사이즈가 기본 제공돼요: 핸들이 포커스된 상태에서 Left/Right(가로) 또는 Up/Down(세로)이 5% 단위로 이동하고, `aria-valuenow`가 갱신돼요.
- 크기는 각 패널의 `minSize` / `maxSize`로 제한되므로, 키보드와 포인터 리사이즈가 패널을 경계 너머로 밀 수 없어요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

각 패널 안에 스크롤 가능한 영역을 두세요 — 패널은 `overflow: auto`를 설정하므로, 줄어든 패널을 초과하는 콘텐츠는 잘리지 않고 스크롤돼요. 가장 중요한 패널은 `minSize` 위로 유지하세요.

## 라이브 데모

React와 Web Components가 아래에서 같은 토큰으로 라이브 렌더돼요 — 어느 쪽 구분선이든 드래그하거나, 포커스한 뒤 화살표 키로 크기를 조정하고, 헤더에서 다크 모드를 토글하면 둘이 동일하게 반응해요. React Native는 추가 `react-native-web` 연결 없이는 브라우저에서 실행할 수 없어요(이 사이트에 설정되지 않음), 따라서 사용법은 코드 샘플로 보여줘요. (React Native에서 핸들은 터치 드래그 전용이에요 — 화살표 키 리사이즈가 없어요.)
