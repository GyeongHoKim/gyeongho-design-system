## 개요

`Message`는 채팅 메시지 행이에요: 선택적 아바타, 그리고 작성자 + 타임스탬프 헤더와 `Bubble`이 있는 콘텐츠 열. `side` prop이 레이아웃을 뒤집어 발신 메시지가 끝에 정렬돼요. `MessageAvatar`, `MessageContent`, `MessageAuthor`, `MessageTimestamp`로 조합하고, `Bubble`(`@ghds/react/bubble`에서)과 `Avatar`(`@ghds/react/avatar`에서)를 함께 쓰세요.

## 구조

다음과 같이 배치된 flex 행이에요: `MessageAvatar`(선행 `Avatar`)와 `MessageContent` — 헤더(`MessageAuthor` + `MessageTimestamp`)를 `Bubble` 위에 담는 열. `side="sent"`는 행을 뒤집어 아바타와 말풍선이 끝에 정렬돼요. 간격과 텍스트 색상은 `@ghds/tokens`(`comp.message.*`)에서 와요.

## 변형 및 상태

유일한 축은 `side`(`received`는 시작에, `sent`는 끝에 정렬)예요. 말풍선이 시각적 변형을 가져요.

## 접근성

- 작성자 이름(`MessageAuthor`)이 화자를 명확히 하므로, 정체성이 아바타나 말풍선 색상만에 의존하지 않아요.
- `Avatar`에 `name`을 제공해 접근 가능한 라벨 / 이니셜 폴백을 갖게 하세요.
- 전체 로그의 경우, 순서가 보조 기술에 전달되도록 메시지를 `MessageScroller`(또는 list/log 랜드마크)로 감싸세요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

헤더는 짧은 작성자 이름과 간결한 타임스탬프로 유지하세요. 메시지 텍스트는 `Bubble`에 있어요; 그 안에 작성자를 반복하지 마세요.

## 라이브 데모

```tsx
// React Native
import { Message, MessageAvatar, MessageContent, MessageAuthor, MessageTimestamp } from '@ghds/react-native/message';
import { Bubble } from '@ghds/react-native/bubble';
import { Avatar } from '@ghds/react-native/avatar';

<Message side="received">
  <MessageAvatar>
    <Avatar name="Bo Ram" />
  </MessageAvatar>
  <MessageContent>
    <MessageAuthor>Bo Ram</MessageAuthor>
    <MessageTimestamp>10:02</MessageTimestamp>
    <Bubble variant="received">Hey — are we still on for tomorrow?</Bubble>
  </MessageContent>
</Message>;
```
