## 개요

데이터를 수집하기 위한 단일 행 텍스트 필드로, 선택적 보이는 라벨과(React에서) 오류 메시지를 가져요.

## 구조

손그림 외곽선의 텍스트 박스 위에 선택적 라벨; React에서는 `error`가 설정되면 박스 아래에 `role="alert"` 요소로 오류 메시지가 렌더돼요.

Web Components는 대신 `required`(`ElementInternals`를 통한 네이티브 제약 검증)를 가져요 — React와 React Native는 그걸 prop으로 노출하지 않아요. 이는 문서 누락이 아니라 두 개의 별개 실제 간극이에요: Web Components나 React Native에 대한 오류 데모를 보여주지 마세요.

## 접근성

- **React**: `aria-invalid` + `aria-describedby`(오류 `<span role="alert">`를 가리킴) + Radix `Label`/`htmlFor` 연결.
- **Web Components**: `label`/`for` 연결; 라벨이 없으면 `aria-label`이 플레이스홀더로 폴백해요. 현재 무효 상태 ARIA는 없어요.
- **React Native**: `accessibilityLabel`이 `placeholder`로 폴백; `accessibilityState={{disabled}}`는 오류/무효 개념이 없어요.
- 일반적인 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

라벨은 수집하는 데이터를 나타내요("Email", "Password"), 동작이 아니에요 — 라벨이 동작을 나타내는 Button과 대조돼요. 오류 메시지는 무엇이 잘못되었고 어떻게 고치는지 말해줘요.
