## 개요

더 긴 자유 형식 데이터를 수집하기 위한 여러 줄 텍스트 필드로, 선택적 보이는 라벨, (React에서) 에러 메시지,
그리고 콘텐츠에 맞게 필드를 늘리는 옵션인 `autoResize` 모드가 있어요. 스크롤 대신 콘텐츠에 맞게 늘어나요.

## 구조

스케치 스타일 윤곽선의 텍스트 박스 위에 선택적 라벨이 있어요; React에서는 `error`가 설정되면
박스 아래에 `role="alert"` 엘리먼트로 에러 메시지가 렌더링돼요. 네이티브 크기 조정 핸들은 항상
비활성화되어 있어요 — 손그림 스타일 테두리와 시각적으로 충돌해서, 박스 크기가 변하려면
(`autoResize` 설정 시) 성장하는 것만이 유일한 방법이에요.

아니에요 — Input과 동일한 플랫폼 분할이에요.

## 접근성

- **React**: `aria-invalid` + `aria-describedby`(에러 `<span role="alert">`를 가리켜요) +
  Radix `Label`/`htmlFor` 연결.
- **Web Components**: `label`/`for` 연결; 라벨이 설정되지 않으면 `aria-label`이 플레이스홀더로
  대체돼요. 현재 무효 상태 ARIA는 없어요.
- **React Native**: `label`은 필수 항목이에요 (RN에는 별도의 라벨 엘리먼트가 없어요)이며
  `accessibilityLabel`을 겸해요; `accessibilityState={{disabled}}`에는 에러/무효 개념이 없어요.
- 일반적인 규칙은 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

라벨은 수집 중인 데이터의 이름이에요 ("자기소개", "피드백"), 동작이 아니에요 — 라벨이 동작의
이름인 Button과 대조적이에요. 에러 메시지는 무엇이 잘못되었는지와 해결 방법을 명시해요.

## 라이브 데모

React 데모에는 에러 상태가 포함되어 있어요; Web Components 데모는 의도적으로 포함하지 않아요
(error prop이 없어요) 대신 `required`를 보여줘요. 두 데모 모두 `autoResize` 예제를 포함해요.
