`ScrollArea`는 **React 전용**이에요(`@ghds/react`). 아직 Web Components나 React Native 대응물은 없어요.

## 개요

`ScrollArea`는 경계 스크롤 뷰포트예요: 네이티브 스크롤바가 디자인 시스템에 맞게 테마된, 스크롤 가능한 영역 주위에 손그림 테두리를 그려요. 넘치는 콘텐츠 — 긴 목록, 코드 블록, 옵션 패널 — 의 높이(또는 너비)를 제한하면서 스케치 느낌을 일관되게 유지할 때 사용하세요.

## 구조

내부 뷰포트 주위의 손그림 테두리(`@ghds/sketch-core`)예요. 뷰포트는 표준 `scrollbar-color` / `scrollbar-width` CSS 속성을 사용하므로, 커스텀 스크롤바 오버레이 없이도 썸과 트랙이 `comp.scrollArea.*` 토큰을 가져와요(Firefox 및 Chromium 121+). `style`을 통해 `maxHeight`(또는 `maxWidth`)를 전달해 영역을 제한하세요.

## 변형 및 상태

인터랙티브 상태가 없어요 — 유일한 축은 스크롤 축을 선택하는 `orientation`이에요.

## 접근성

- 스크롤되는 영역은 키보드로 도달할 수 있어야 해요. 콘텐츠가 자연스럽게 포커스 가능하지 않을 때, 키보드 사용자가 스크롤할 수 있도록 뷰포트에 `tabIndex={0}`과 `aria-label`을 지정하세요.
- 테마된 스크롤바는 표준 `scrollbar-*` 속성을 사용하므로 네이티브 스크롤바로 우아하게 저하돼요 — 스크롤 어포던스를 완전히 숨기지 않아요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.
