## 개요

`Checkbox`는 켜기/끄기 불리언 입력이에요 — `indeterminate`("일부만 선택됨") 상태도 보여줄 수 있어요. `CheckboxGroup`은 관련 체크박스들을 묶고 결합된 값을 배열로 관리해요.

## 구조

실제 `<input type="checkbox">`(React/Web Components) 또는 `accessibilityRole="checkbox"`를 가진 `Pressable`(React Native) 뒤에 그려진 손그림 박스예요. 체크되거나 중간 상태일 때 손그림 체크 또는 마이너스 표시가 위에 렌더돼요 — 새로운 글리프 형상을 만드는 대신 기존 `Icon`/`gh-icon` 컴포넌트를 재사용해요.

## 접근성

- **React**: 실제 `<input type="checkbox">`가 네이티브 키보드(Space 토글)와 포커스 동작을 가져요. `indeterminate`는 DOM 전용 프로퍼티(HTML 속성이나 폼 값이 아님)예요 — 네이티브 브라우저/다른 체크박스 구현처럼 기본 input에 명령형으로 설정해요.
- **Web Components**: `gh-checkbox`는 섀도 DOM 안에 실제 `<input type="checkbox">`를 렌더해서 네이티브 동작을 제공하고, `ElementInternals`를 통해 실제 `<form>` 참여로 체크 상태를 연결해요(폼 값은 체크됐을 때만 존재). 또한 reflected된 `checked`/`indeterminate` 속성과 함께 `ElementInternals.states`를 통해 `:state(checked)`/`:state(indeterminate)` CSS 의사 클래스도 노출해요.
- **React Native**: 네이티브 플랫폼을 위해 `accessibilityRole="checkbox"` + `accessibilityState.checked`(중간 상태일 때 `'mixed'`), 그리고 웹 타겟을 위한 직접 `aria-checked` prop(React Native Web은 `accessibilityState`에서 `aria-checked`를 `aria-disabled`처럼 파생하지 않아요).
- 일반적인 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.
