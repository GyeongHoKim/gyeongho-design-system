## 개요

`Card`는 관련 콘텐츠를 묶기 위한 범용 표면이에요 — 자체적으로 인터랙티브하지 않고 "이것들이 함께 속한다"는 의미 외에는 내장된 의미가 없어요.

## 구조

일반 자식 요소 뒤에 그려진 손그림 사각형 외곽선이에요. Web Components에서는 `<h1>`/`<h2>`/`<h3>`/`[slot="title"]`로 슬롯된 콘텐츠가 특별한 제목 스타일(`::slotted` CSS)을 받아요; React와 React Native에는 동등한 제목 슬롯이 없어요 — 콘텐츠는 그냥 일반 자식 요소예요.

## 변형 및 상태

Button이나 Input과 달리 Card는 주로 **정적 컨테이너**예요 — 이 섹션은 존재하지 않는 상태를 억지로 끼워맞추지 않고 의도적으로 간결해요.

<StateList
  states={[
    { name: 'Hover', present: false, note: '카드는 인터랙티브하지 않아요.' },
    { name: 'Focus', present: false, note: '카드는 포커스 가능하지 않아요.' },
    { name: '비활성화', present: false },
    { name: '로딩', present: false },
    { name: '오류', present: false },
  ]}
/>

유일한 실제 변형은 **플랫폼마다 다른 prop**이지 공통 상태가 아니에요: React의 `fill`(`'solid' | 'hachure'`)이 채우기 패턴을 제어하고; Web Components와 React Native는 대신 `elevated`(불리언 드롭 섀도우 효과)를 노출해요 — React의 상위 집합이 아니라 진짜로 다른 API예요.

## 접근성

- **React**: 일반 `<div>`를 렌더해요. role이나 라벨이 자동으로 설정되지 않아요 — "어떤 ARIA role/랜드마크든 호출자가 설정해요"(컴포넌트의 문서 주석에 따름).
- **Web Components**: `label`이 제공되면 `gh-card`가 자동으로 `internals.role = 'group'`과 `internals.ariaLabel`을 설정해요 — React가 *복제하지 않는* 실제 동작이에요.
- **React Native**: `accessibilityLabel`이 전달될 때만 `accessibilityRole="summary"`를 설정해요.
- 일반적인 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

해당 없음 — 카드는 고정된 콘텐츠 모델이 없어요. 콘텐츠는 호출자가 자식(React/RN)이나 슬롯(Web Components)으로 전달하는 것이에요.

## 라이브 데모

React는 `fill`을 사용하고; Web Components는 `elevated` + `label`을 사용해요 — 아래 데모는 통합된 예시가 아니라 각 플랫폼의 *실제* prop을 의도적으로 보여줘요.
