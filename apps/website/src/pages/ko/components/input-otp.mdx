`InputOTP`는 현재 **React 전용** 컴포넌트예요(`@ghds/react`). 아직 Web Components나 React Native 대응물은 없어요.

## 개요

`InputOTP`는 일회용 코드를 위한 분할형 필드예요 — 하나의 왼쪽 채움 문자열 값으로 뒷받침되는 단일 문자 셀의 행이에요. 이메일이나 SMS로 전송되는 인증 코드와 인증기(TOTP) 입력에 사용하세요. 타이핑하면 셀 사이를 진행하고, 백스페이스는 마지막 문자를 삭제하며, 코드를 붙여넣으면 셀 전체에 분배돼요.

## 구조

`length`개의 셀을 감싸는 라벨이 지정된 `role="group"`이에요. 각 셀은 자체 스케치 `SketchSurface` 박스 안의 바 단일 문자 `<input>`이에요: 활성 셀은 포커스 스트로크를, 채워진 셀은 더 강한 스트로크를 보여주고, invalid 필드는 모든 셀을 danger 스트로크로 그려요. 모든 색상, 크기, 스케치 파라미터는 `@ghds/tokens`(`comp.inputOtp.*`)에서 와요.

## 변형 및 상태

명명된 변형은 없어요 — 대신 `length`, `mode`, `mask`가 필드를 형성하고, `disabled` / `invalid`가 상태를 설정해요. 셀은 또한 활성(포커스)과 채워짐에 반응해요.

## 접근성

- 셀은 `label`(Radix `Label`을 통해) 또는 `aria-label`로 이름 지어진 `role="group"`으로 감싸지므로, 보조 기술이 이를 하나의 필드로 알려요.
- 각 셀은 자체 위치 `aria-label`(`Digit 1`, `Digit 2`, …)을 가지고, 내부 입력은 `autoComplete="one-time-code"`를 사용해 모바일 OS가 수신된 코드를 제안할 수 있어요.
- 키보드 지원이 기본 제공돼요: 타이핑은 진행하고, 백스페이스는 마지막 문자를 삭제하며, Arrow/Home/End는 셀 사이를 이동해요.
- 이유가 색상만이 아니라 설명되도록 `invalid`를 외부 오류 메시지와 함께 사용하세요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

자유 텍스트가 없어요 — 콘텐츠는 코드 자체예요. 함께 붙는 라벨은 짧게(`Verification code`) 유지하고, "재전송" 어포던스나 오류 메시지는 필드 밖에 두세요.
