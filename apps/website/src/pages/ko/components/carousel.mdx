`Carousel`과 그 하위 컴포넌트는 **React 전용**이에요(`@ghds/react`). 아직 Web Components나 React Native 대응물은 없어요.

## 개요

`Carousel`은 가로(또는 세로)로 스크롤되는 슬라이드의 슬라이드쇼예요. 서드파티 엔진 없이 네이티브 CSS 스크롤 스냅으로 만들어졌어요. 한 번에 한 뷰포트씩 동등한 항목 집합을 탐색할 때 사용하세요: 사진 갤러리, 기능 카드 세트, 온보딩 시퀀스. `CarouselContent`, `CarouselItem`, `CarouselPrevious`, `CarouselNext`로 조합하세요.

## 구조

캐러셀로 이름 지어진 `role="region"`이 슬라이드(`CarouselItem`, 각각 슬라이드로 설명된 `role="group"`)의 스크롤 스냅 트랙(`CarouselContent`)을 감싸요. `CarouselPrevious` / `CarouselNext`는 한 스텝에 한 뷰포트씩 이동하고 각각 시작 / 끝에서 비활성화되는 `Button`이에요. 화살표 키가 방향 축을 따라 스크롤하고, 스냅은 네이티브로 유지되면서 스크롤바는 시각적으로 숨겨져요.

## 변형 및 상태

유일한 변형 축은 `orientation`(`'horizontal'` 기본값, 또는 `'vertical'`)이에요. 컨트롤 상태는 스크롤 위치에서 나와요.

## 접근성

- 루트는 ARIA APG 캐러셀 패턴이에요: `role="region"` + `aria-roledescription="carousel"`, `aria-label`로 이름 지어져요. 각 슬라이드는 `aria-roledescription="slide"`가 있는 `role="group"`이에요.
- 컨트롤은 "Previous slide" / "Next slide"로 라벨이 지정된 실제 `<button>`이며 각 끝에서 `disabled`가 되므로 상태가 알려져요.
- 화살표 키(가로일 때 Left/Right, 세로일 때 Up/Down)가 영역을 스크롤하고, 네이티브 스크롤 컨테이너는 키보드와 터치로 조작 가능하게 유지돼요.
- 공통 관례는 <a href={withBase('/accessibility/')}>접근성 가이드</a>를 참고하세요.

## 콘텐츠

슬라이드는 대략 동등한 비중의 동료여야 해요. 각 슬라이드를 자족적으로 유지하고 — 사용자가 어느 것이든 먼저 볼 수 있어요 — 하나의 메시지를 여러 슬라이드에 나누지 마세요.

```tsx
import { Carousel, CarouselContent, CarouselItem, CarouselPrevious, CarouselNext } from '@ghds/react/carousel';

<Carousel aria-label="Photos">
  <CarouselContent>
    <CarouselItem>{/* slide 1 */}</CarouselItem>
    <CarouselItem>{/* slide 2 */}</CarouselItem>
  </CarouselContent>
  <CarouselPrevious />
  <CarouselNext />
</Carousel>;
```
